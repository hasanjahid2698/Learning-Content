1. Encapsulation üõ°Ô∏è

Encapsulation is all about bundling the data (attributes) and the methods that operate on that data into a single unit called a class. It restricts direct access to an object's internal state, hiding implementation details from the outside world.

  • Key Idea: Protect an object's internal state from unintended external modification.
  • How: Make attributes private and provide public methods (getters and setters) to access and modify them.

Python Example:

class BankAccount: def __init__(self, account_number, initial_balance): self.__account_number = account_number # Private attribute self.__balance = initial_balance # Private attribute def deposit(self, amount): if amount > 0: self.__balance += amount def withdraw(self, amount): if 0 < amount <= self.__balance: self.__balance -= amount def get_balance(self): return self.__balance # Usage my_account = BankAccount("12345", 1000) # print(my_account.__balance) # This will cause an AttributeError print(my_account.get_balance()) # Correct way to access the balance

2. Inheritance üë®‚Äçüë©‚Äçüëß

Inheritance allows a class (the child or subclass) to inherit attributes and methods from another class (the parent or superclass). This promotes code reuse and establishes a hierarchical relationship between classes.

  • Key Idea: "Is-a" relationship (e.g., a Car is a Vehicle). Reuse code from a parent class.
  • How: Create a new class that derives from an existing class.

Python Example:

class Vehicle: # Parent class def __init__(self, color): self.color = color def honk(self): print("Honk honk!") class Car(Vehicle): # Child class def __init__(self, color, speed): super().__init__(color) # Call parent's constructor self.speed = speed def accelerate(self): print("Vroom!") # Usage my_car = Car("Red", 120) print(my_car.color) # Inherited attribute my_car.honk() # Inherited method my_car.accelerate() # Own method

3. Polymorphism üé≠

Polymorphism, meaning "many forms," is the ability of an object to take on multiple forms. It allows you to write generic code that can work with objects of different types as long as they share a common interface.

  • Key Idea: Use a single interface to represent different underlying forms (data types).
  • How: A subclass provides its own specific implementation of a method that is already defined in its parent class.

Python Example:

class Document: def show(self): raise NotImplementedError class Pdf(Document): def show(self): return "Showing a PDF document." class Word(Document): def show(self): return "Showing a Word document." # Usage documents = [Pdf(), Word()] for doc in documents: # The same method call works for different objects print(doc.show())

4. Abstraction ü§´

Abstraction focuses on hiding complex implementation details while showing only the necessary information to the user. It helps to simplify complex systems by modeling classes appropriate to the problem.

  • Key Idea: Expose what an object does, not how it does it.
  • How: Use abstract base classes (ABCs) and abstract methods to define a required interface for subclasses.

Python Example:

from abc import ABC, abstractmethod class Shape(ABC): # Abstract base class @abstractmethod def area(self): pass class Rectangle(Shape): def __init__(self, width, height): self.width = width self.height = height def area(self): # Provides implementation return self.width * self.height class Circle(Shape): def __init__(self, radius): self.radius = radius def area(self): # Provides implementation return 3.14 * self.radius ** 2 # Usage rect = Rectangle(10, 5) circ = Circle(7) print(f"Rectangle Area: {rect.area()}") print(f"Circle Area: {circ.area()}")
